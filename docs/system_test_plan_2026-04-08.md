# 全系统测试计划

日期：2026-04-08  
执行人：Codex

## 目标

- 验证核心服务可启动、可联通、不会因关键请求直接崩溃。
- 验证前后端主流程可用，避免“页面打不开、接口 500、关键业务无法推进”。
- 对 AI、工单、客户、报价、门店总览等关键链路做回归。
- 输出失败项、风险项和建议补强项。

## 测试范围

### 一、基础可用性

- Docker 服务状态检查
- `bff` 健康检查
- `ai` 健康检查
- `odoo` 容器在线状态
- 数据库、Redis、MinIO、OCR 容器在线状态

### 二、后端自动化

- 运行 `bff/tests` 下 pytest 用例
- 核查启动校验、文本支持、AI Ops、API 基本行为

### 三、前端构建稳定性

- `clients/web_staff`
- `clients/cs_workspace`
- `clients/web_display`
- `clients/mp_customer`
- `clients/mp_staff`
- `clients/mp_customer_uni`

### 四、关键业务集成冒烟

- 登录鉴权
- 客户创建
- 工单创建
- 工单文档渲染
- 工单动作接口
- 报价创建与发布
- 门店总览接口
- AI 对话查询
- AI 录入/修改动作

### 五、稳定性与异常观察

- 关键接口是否出现 500
- 是否存在明显超时
- 是否存在中文编码、字段缺失、空上下文导致的崩溃

## 执行顺序

1. 服务基线检查
2. 后端自动化测试
3. 前端全量构建
4. 关键业务冒烟
5. AI 主链路回归
6. 问题修复与回归

## 通过标准

- 核心服务全部在线
- 后端自动化测试无阻断级失败
- 主要前端项目可完成生产构建
- 客户、工单、报价、AI 查询与 AI 写入至少各有一条真实链路通过
- 无新增致命错误或明显死机风险
